Simulación de ataques para SecureMonitor

Usa el endpoint interno /api/simulate para generar tráfico malicioso controlado y ver cómo reacciona el dashboard en tiempo real.

Endpoint listo para pruebas internas

Este módulo no abre ningún servicio real, solo inyecta eventos al daemon.

1. Endpoint base

El endpoint acepta método POST o GET y recibe dos parámetros:

  • kind: ssh | ftp | apache
  • n: cantidad de eventos a simular (1–1000, por defecto 10)
Forma general
/api/simulate?kind=ssh&n=20 /api/simulate?kind=ftp&n=50 /api/simulate?kind=apache&n=15

2. Ejemplos con curl (desde tu terminal)

SSH – múltiples intentos de login fallidos

curl -X POST "https://securemonitor.juvenr.com/api/simulate?kind=ssh&n=15"

FTP – autenticaciones fallidas en vsftpd

curl -X POST "https://securemonitor.juvenr.com/api/simulate?kind=ftp&n=25"

Apache – errores 4xx/5xx desde tu IP

curl -X POST "https://securemonitor.juvenr.com/api/simulate?kind=apache&n=12"

3. Qué pasa detrás

  • SecureMonitor toma tu IP desde X-Forwarded-For / RemoteAddr.
  • Inyecta los eventos simulados en los contadores internos (SSH/FTP/Apache).
  • El RunLoop procesa esos eventos como si vinieran de los logs reales.
  • Se generan alertas, se actualizan stats y, si pasas el umbral, se puede bloquear tu IP vía UFW.
