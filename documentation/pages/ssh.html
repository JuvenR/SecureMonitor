SSH en Linux - OpenSSH Server

Documentación del proceso de instalación, configuración y uso del servicio SSH (Secure Shell) en una máquina con Ubuntu, incluyendo conexión remota desde Windows mediante CMD.

Concepto

¿Qué es el servicio SSH?

SSH (Secure Shell) permite conectarse y administrar equipos de manera remota a través de una conexión segura y cifrada.

En Linux, este servicio se implementa mediante el paquete OpenSSH Server, el cual crea un canal seguro entre un cliente (quien se conecta) y un servidor (el equipo remoto).

Con SSH puedes:

  • Acceder a una terminal remota y ejecutar comandos.
  • Transferir archivos con SCP o SFTP.
  • Crear túneles seguros para otros servicios.
  • Administrar servidores de manera remota sin interfaz gráfica.

SSH es fundamental para la administración remota de servidores Linux, proporcionando seguridad mediante cifrado de toda la comunicación.

Instalación

Pasos para levantar el servicio SSH en Linux

Proceso completo de instalación y configuración del servicio OpenSSH Server.

1. Actualizar el sistema

Antes de instalar cualquier paquete, se recomienda actualizar la lista de paquetes:

Comando de actualización
sudo apt update && sudo apt upgrade -y
Ejecución en terminal

2. Instalar el servicio SSH

En la mayoría de las distribuciones modernas (Ubuntu, Debian, Fedora, CentOS, etc.) el paquete se llama openssh-server.

Comando de instalación
sudo apt install openssh-server -y
Ejecución en terminal

3. Iniciar y habilitar el servicio

Una vez instalado, se debe activar el servicio SSH para que inicie automáticamente al encender el sistema:

Habilitar e iniciar SSH
sudo systemctl enable ssh sudo systemctl start ssh
Ejecución en terminal
carlos@Host:~$ sudo systemctl enable ssh [sudo] password for carlos: Synchronizing state of ssh.service with SysV service script with /usr/lib/systemd/systemd-sysv-install. Nota: Durante la instalación se muestra un mensaje sobre "cache lock" porque otro proceso (unattended-upgr) estaba usando el gestor de paquetes. Esto es normal y el sistema espera a que se libere el recurso.

Verificación

Verificar el estado del servicio

Comprobar que el servicio esté activo y funcionando correctamente.

Para verificar el estado del servicio SSH, se utiliza el comando:

Configuración

Identificar la IP del servidor

Obtener la dirección IP del servidor SSH para permitir conexiones remotas.

Para conocer la dirección IP asignada al servidor, se utiliza el comando:

Comando para ver interfaces de red
ip a
Salida del comando
carlos@Host:~$ ip a 1: lo: mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000 link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 inet 127.0.0.1/8 scope host lo valid_lft forever preferred_lft forever inet6 ::1/128 scope host noprefixroute valid_lft forever preferred_lft forever 2: enp0s3: mtu 1500 qdisc fq_codel state UP group default qlen 1000 link/ether 08:00:27:3a:46:3e brd ff:ff:ff:ff:ff:ff inet 192.168.4.110/24 brd 192.168.4.255 scope global dynamic noprefixroute enp0s3 valid_lft 86295sec preferred_lft 86295sec inet6 fe80::a00:27ff:fe3a:463e/64 scope link valid_lft forever preferred_lft forever

En este caso, la dirección IP del servidor es: 192.168.4.110

La interfaz enp0s3 tiene asignada la IP 192.168.4.110/24, lo que indica que está en la red 192.168.4.0 con máscara 255.255.255.0.

Conexión

Conexión remota desde otro dispositivo

Establecer conexión SSH desde Windows CMD hacia el servidor Linux.

Para conectarse desde otro dispositivo (en este caso, CMD de Windows), se debe:

  1. Cambiar el adaptador de red de NAT a Adaptador puente en la configuración de la VM
  2. Usar el comando SSH con la sintaxis: ssh usuario@ip_del_servidor

Práctica

Experimento práctico con SSH

Ejecución de comandos remotos para demostrar el funcionamiento de SSH.

Una vez conectado por SSH, se realizó un experimento para demostrar la capacidad de ejecutar comandos de manera remota:

Procedimiento:

1. Crear carpeta
mkdir prueba
2. Entrar a la carpeta
cd prueba
3. Crear archivo con contenido
echo "Hola soy yo desde el cmd" > archivo.txt
4. Leer el contenido del archivo
cat archivo.txt

Resultado esperado:

Hola soy yo desde el cmd

Este experimento demuestra que a través de SSH se pueden ejecutar comandos de administración del sistema de forma remota, incluyendo la creación y manipulación de archivos y directorios.
